# Deployment Guide — payroll-web

_Generated 2026-05-25 — v0.0.0_
This document describes the deployment process, environments, and release workflow for **payroll-web**, a web-based payroll management system built with React, Vite, and Firebase.

---

## 🏗️ Environments

Our project utilizes distinct Firebase projects for different deployment environments to ensure isolation and proper testing before production releases.

| Environment     | Associated Firebase Project ID | Branch       | Typical URL                               |
| :-------------- | :----------------------------- | :----------- | :---------------------------------------- |
| **Development** | `payroll-web-dev`              | `main`       | `localhost:5001` (local)                  |
| **Staging**     | `payroll-web-staging`          | `staging`    | `payroll-web-staging.web.app` or `staging.payroll-web.com` |
| **Production**  | `payroll-web`                  | `production` | `payroll-web.web.app` or `payroll-web.com` |

Each Firebase project (`payroll-web-dev`, `payroll-web-staging`, `payroll-web`) manages its own set of services including Hosting, Firestore Database, and Firebase Authentication.

## 🛠️ Prerequisites

Before you can deploy `payroll-web`, ensure you have the following installed and configured:

1.  **Node.js**: Version 18 or higher.
2.  **Yarn**: Our preferred package manager.
    ```bash
    npm install -g yarn
    ```
3.  **Firebase CLI**: The command-line interface for Firebase.
    ```bash
    npm install -g firebase-tools
    ```
4.  **Firebase Account & Projects**:
    *   A Google account with access to Firebase.
    *   Three Firebase projects created and configured: `payroll-web-dev`, `payroll-web-staging`, and `payroll-web`.
    *   You must be logged into the Firebase CLI:
        ```bash
        firebase login
        ```
    *   Ensure your local project is linked to these Firebase projects using aliases in your `.firebaserc` file (see Project Configuration). If not already configured, run `firebase init` and select "Firebase Hosting" and "Firestore" to generate the initial configuration files.

## ⚙️ Project Configuration

### `.firebaserc` File

This file in the project root defines aliases for your Firebase projects, allowing you to easily switch between them using `firebase use`.
An example `firebaserc` for this project:

```json
{
  "projects": {
    "default": "payroll-web-dev",
    "dev": "payroll-web-dev",
    "staging": "payroll-web-staging",
    "production": "payroll-web"
  }
}
```

To switch to a specific project for CLI commands:
```bash
firebase use staging
# Output: Active project set to payroll-web-staging
```
Or for production:
```bash
firebase use production
# Output: Active project set to payroll-web
```

### `firebase.json` File

This file configures Firebase Hosting, Firestore rules, and potentially other Firebase services. For `payroll-web`, it primarily focuses on hosting static assets and configuring rewrites for React Router.

```json
{
  "hosting": {
    "public": "dist",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**"
    ],
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ]
  },
  "firestore": {
    "rules": "firestore.rules",
    "indexes": "firestore.indexes.json"
  }
}
```
*   `hosting.public`: Specifies that our built React application (generated by Vite) will be in the `dist` directory.
*   `hosting.rewrites`: Ensures that all routes are handled by our `index.html` file, which is crucial for client-side routing with React Router.
*   `firestore.rules`: Points to the file containing Firestore security rules.
*   `firestore.indexes`: Points to the file defining custom Firestore indexes.

### Environment Variables

`payroll-web` uses Vite's built-in environment variable support. Variables prefixed with `VITE_` are exposed to your client-side code. These are typically defined in `.env` files in your project root (e.g., `.env.development`, `.env.production`).

Example `.env` file for development (not committed to Git):
```
VITE_FIREBASE_API_KEY=your-dev-api-key
VITE_FIREBASE_AUTH_DOMAIN=your-dev-auth-domain
VITE_FIREBASE_PROJECT_ID=payroll-web-dev
# ... other Firebase config variables
```

**Important**: Environment variables for the client-side app are **baked into the build**. To deploy to a different environment, you must rebuild the application with the correct `.env` variables loaded (e.g., by ensuring the correct `.env.production` or similar file is present during the `yarn build` step, or by passing them directly during the build command in a CI/CD pipeline).

## 🚀 Deployment Workflow

This section outlines the steps to deploy `payroll-web` to our different environments.

### 1. Local Development

To run the application locally for development:
```bash
yarn install
yarn dev
```
This will start the Vite development server, typically accessible at `http://localhost:5173` (or similar). The Firebase emulator suite can also be used for local testing of Firebase services.

### 2. Building the Application

Before any deployment, you must build the frontend application. This compiles your React code, optimizes assets, and places them into the `dist` directory.
```bash
yarn build
```
This command ensures that all production-ready assets are generated.

### 3. Running Tests

Always run the full test suite before deployment to ensure no regressions have been introduced.
```bash
yarn test
```
Ensure all tests pass successfully.

### 4. Deploying to Staging

The staging environment (`payroll-web-staging` Firebase project) is used for integration testing, QA, and stakeholder review before pushing to production.

1.  **Switch to the Staging Firebase project:**
    ```bash
    firebase use staging
    ```
2.  **Ensure correct environment variables are used for the build.** If you manage environment variables through `.env` files, ensure your `.env.staging` (or equivalent) is correctly configured and loaded during the build process.
3.  **Build the application for staging:**
    ```bash
    yarn build
    ```
4.  **Deploy to Firebase Hosting:**
    ```bash
    firebase deploy --only hosting
    ```
    This command uploads the contents of your `dist` folder to Firebase Hosting for the `payroll-web-staging` project.
5.  **Deploy Firestore Security Rules (if updated):**
    If you've made changes to `firestore.rules`, deploy them:
    ```bash
    firebase deploy --only firestore:rules
    ```
6.  **Deploy Firestore Indexes (if updated):**
    If you've made changes to `firestore.indexes.json`, deploy them:
    ```bash
    firebase deploy --only firestore:indexes
    ```
7.  **Verify Deployment:** Access the staging URL (e.g., `payroll-web-staging.web.app`) to ensure the application is running as expected.

### 5. Deploying to Production

The production environment (`payroll-web` Firebase project) hosts the live application for end-users. This step should only be performed after thorough validation in staging.

1.  **Switch to the Production Firebase project:**
    ```bash
    firebase use production
    ```
2.  **Ensure correct environment variables are used for the build.** Verify that your `.env.production` (or equivalent) is correctly configured and loaded.
3.  **Build the application for production:**
    ```bash
    yarn build
    ```
4.  **Deploy to Firebase Hosting:**
    ```bash
    firebase deploy --only hosting
    ```
    This command uploads the production-ready build to Firebase Hosting for the `payroll-web` project.
5.  **Deploy Firestore Security Rules (if updated):**
    ```bash
    firebase deploy --only firestore:rules
    ```
6.  **Deploy Firestore Indexes (if updated):**
    ```bash
    firebase deploy --only firestore:indexes
    ```
7.  **Verify Deployment:** Access the production URL (e.g., `payroll-web.com`) to confirm the successful deployment.

## 📦 Release Process (Summary)

This is the high-level release flow for `payroll-web`:

1.  **Develop Features**: Work on features in dedicated branches off `main`.
2.  **Merge to `main`**: Once features are complete and reviewed, merge them into the `main` branch.
3.  **Run Full Test Suite**: Execute `yarn test` to ensure stability.
4.  **Build Application**: Run `yarn build` with staging-specific environment variables.
5.  **Deploy to Staging**:
    ```bash
    firebase use staging && yarn build && firebase deploy --only hosting,firestore:rules,firestore:indexes
    ```
6.  **Validate on Staging**: QA team and stakeholders thoroughly test the application on the staging environment.
7.  **Tag Release**: Upon successful validation, create a Git tag for the release (e.g., `git tag v1.0.0`).
8.  **Deploy to Production**:
    ```bash
    firebase use production && yarn build && firebase deploy --only hosting,firestore:rules,firestore:indexes
    ```
    (Ensure production environment variables are used for the build.)
9.  **Monitor Production**: Observe application performance and error logs post-deployment.

## 🐳 Containerization

Containerization (e.g., Docker) is not configured for `payroll-web`. Given our tech stack (React frontend served by Firebase Hosting, Firebase for backend services like Firestore and Auth), Docker is generally not required. Firebase Hosting effectively handles the "containerization" and scaling of our static frontend assets. If Firebase Cloud Functions were heavily used, Docker might be considered for those, but for the current setup, it adds unnecessary complexity.
